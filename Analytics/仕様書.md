## 2. 新規プロジェクトへの適用手順

### 2-1. プロジェクトディレクトリを作成する

案件番号ベースのフォルダ（例: `BA01-XXX-XX_分析名/`）を作成し、その中に `#work/<担当者>/` などのサブディレクトリを用意する。

### 2-2. テンプレートを複製する

`汎用/#work/` 以下のファイル一式をプロジェクトの `#work/<担当者>/` にコピーする。

```
コピー対象:
  configs/
  funcs/
  notebooks/          ← フォルダごとコピー（templates/ 含む）
  requirements.txt
```

### 2-3. configs.yaml を編集する

```yaml
project:
  name: '分析名'
  id: 'BA01-XXX-XX'
  description: '分析の概要'

paths:
  data: '../data'
  processed_data: '../processed_data'
  figures: '../results/figures'
  tables: '../results/tables'

period:
  start: '202301'
  end:   '202312'
  train_end:  '202309'   # モデル構築がある場合
  test_start: '202310'   # モデル構築がある場合

random_seed: 42
```

### 2-4. 仮想環境・ライブラリをセットアップする

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### 2-5. ノートブックを作成する

`notebooks/templates/` から対応するテンプレートを作業フォルダにコピーしてリネームする。

```
例: 01_data_check.ipynb のコピーと配置
  コピー元: notebooks/templates/01_data_check.ipynb
  配置先:   notebooks/01_data/01_data_check.ipynb   （そのまま使用開始）
            notebooks/01_data/01_check_追加データ.ipynb （同じテンプレートから作る場合）
```

> **templates/ は編集しない**。作業はすべて各フェーズフォルダ内のコピーに対して行う。

### 3-2. 可視化ノートブックでの設定値の展開方法

`visualize.yaml` の設定を `visualization.setup_matplotlib()` でmatplotlibに適用するだけでなく、**返り値の `viz_cfg` から設定値を変数として展開**し、各プロットで参照する。こうすることで `visualize.yaml` を1か所変更するだけで全グラフに反映される。

#### セットアップセルの書き方

```python
cfg     = data_loader.load_yaml('../../configs/configs.yaml')
viz_cfg = visualization.setup_matplotlib('../../configs/visualize.yaml', theme='presentation')
# ↑ rcParams をグローバルに適用（フォントサイズ・グリッド・線幅など）

# --- 設定値を変数として展開 ---
_mpl_cfg   = viz_cfg.get('visualization', {}).get('matplotlib', {})
_theme_key = viz_cfg.get('visualization', {}).get('theme', 'presentation')
_theme_cfg = viz_cfg.get('theme', {}).get(_theme_key, {})
_sns_cfg   = viz_cfg.get('visualization', {}).get('seaborn', {})
_pal_cfg   = viz_cfg.get('palette', {})

FIGSIZE      = tuple(_mpl_cfg.get('figure', {}).get('figsize', [12, 6]))  # 標準サイズ
FIGSIZE_WIDE = FIGSIZE                                                      # 横長（同値）
FIGSIZE_SQ   = (FIGSIZE[1], FIGSIZE[1])                                    # 正方形（散布図・相関）
DPI          = _mpl_cfg.get('figure', {}).get('dpi', 150)

COLORS   = _pal_cfg.get('categorical',
           ['#4C72B0','#DD8452','#55A868','#C44E52','#8172B2',
            '#937860','#DA8BC3','#8C8C8C','#CCB974','#64B5CD'])
CMAP_SEQ = _pal_cfg.get('sequential', 'Blues')   # 連続値ヒートマップ
CMAP_DIV = _pal_cfg.get('diverging',  'RdBu_r')  # ダイバージングヒートマップ
PALETTE  = _sns_cfg.get('palette', 'deep')        # seaborn パレット

LW    = _mpl_cfg.get('lines', {}).get('linewidth', 2)
ALPHA = _mpl_cfg.get('scatter', {}).get('alpha', 0.7)
```

#### 各プロットでの参照例

```python
# ✗ 設定値をハードコードしている例
fig, ax = plt.subplots(figsize=(14, 5))
ax.plot(x, y, color='steelblue', linewidth=2)

# ✓ 設定変数を参照している例（visualize.yaml を変えれば全グラフに反映）
fig, ax = plt.subplots(figsize=FIGSIZE_WIDE, dpi=DPI)
ax.plot(x, y, color=COLORS[0], linewidth=LW)
```

| 変数 | 型 | 内容 |
|------|----|------|
| `FIGSIZE` | `tuple` | 標準の図サイズ（例: `(12, 6)`） |
| `FIGSIZE_SQ` | `tuple` | 正方形サイズ（散布図・相関行列用） |
| `DPI` | `int` | 解像度（保存時に使用） |
| `COLORS` | `list[str]` | カテゴリ別カラーリスト（10色） |
| `CMAP_SEQ` | `str` | 連続値ヒートマップのカラーマップ名 |
| `CMAP_DIV` | `str` | 差分・変化率ヒートマップのカラーマップ名 |
| `LW` | `float` | 折れ線グラフの線幅 |
| `ALPHA` | `float` | 散布図の透明度 |

### 3-3. データ確認ノートブックのチェック項目

`01_data_check.ipynb` は受領データチェックシート（Excel）の8カテゴリに準拠した構成になっている。

| セクション | 主なチェック内容 |
|-----------|---------------|
| **1. 基本情報** | ファイル一覧・サイズ・形式・数 |
| **2. データ構造** | レコード数・カラム数・主キー重複・時系列の月別網羅性（欠落月の自動検出） |
| **3. 欠損値** | 欠損値サマリー・必須列チェック・欠損パターンのヒートマップ |
| **4. データ型** | 文字列として読まれた数値の自動検出・日付変換失敗件数・コード桁数バリエーション |
| **5. 異常値** | IQR外れ値の全列スキャン・負の値確認・ドメイン知識による不正値チェック |
| **6. 重複** | 主キー重複・完全一致行 |
| **7. 論理整合性** | マスタ参照チェック・日付前後関係・金額整合性（単価×数量） |
| **8. メタ情報・セキュリティ** | データ辞書との照合・個人情報列名パターンによる自動検出 |
| **9. 確認結果サマリー** | 上記の主要指標を1セルで一覧出力 |

ノートブック先頭のMarkdownセルにチェックリスト表（☐/☑）が設置されており、確認後に随時更新する。


#### カラーパレット設定（`palette` セクション）

`visualize.yaml` の `palette` セクションで色を定義し、ノートブックで `COLORS` 変数として展開して使う。

```yaml
palette:
  categorical:  # カテゴリ変数の色分け（リスト形式、10色）
    - '#4C72B0'
    - '#DD8452'
```
